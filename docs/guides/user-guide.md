## 1. 일상 워크플로우

### 1.1 하루 시작

```bash
# claude-life 폴더에서
cd ~/Develop/claude-life
claude
```

그 다음:
```
나: 오늘 뭐 해야 해?
또는
나: 브리핑
```

→ 오늘 할 일, 긴급한 것, 진행 중인 것 요약해서 보여줌

### 1.2 프로젝트 작업

```bash
# 프로젝트 폴더에서
cd ~/Develop/my-project
claude
```

→ CLAUDE.md에 claude-life 참조가 있으면 **자동으로 컨텍스트 로드**
→ 별도 명령 없이 바로 작업 시작 가능

### 1.3 작업 중 기록

작업하다가 기록할 게 있으면 그냥 자연어로:

```
나: 로그인 API 연동 완료했어. 토큰 갱신 로직은 내일 해야함

나: 버그 발견 - 세션 만료 시 크래시 남. 내일까지 고쳐야 해

나: Riverpod 상태관리 배웠는데 Provider보다 타입 안전하네
```

→ Claude가 알아서 분류해서 저장:
- 태스크 → `20-tasks/`
- 지식 → `30-areas/`
- 일일 기록 → `40-logs/daily/`

### 1.4 하루 끝

```
나: 오늘 작업 정리해줘
```

→ 일일 로그 생성/업데이트, 프로젝트 노트 갱신

## 2. 새 프로젝트 시작할 때

### 2.1 프로젝트 폴더 생성 (평소처럼)

```bash
cd ~/Develop/projects
mkdir new-app
cd new-app
git init
```

### 2.2 Claude-Life에 등록

**방법 A: 자동 동기화 (권장)**

```bash
cd ~/Develop/claude-life
claude
```

```
나: 프로젝트 동기화 해줘
또는
나: /sync
```

→ ~/Develop/* 전체 스캔해서 새 프로젝트 자동 등록

**방법 B: 수동 등록**

```
나: new-app 프로젝트 등록해줘. flutter 프로젝트야.
```

### 2.3 프로젝트에 CLAUDE.md 생성

`/sync` 실행 시 CLAUDE.md 없는 프로젝트 발견하면 생성 제안함.

또는 수동으로:

```
나: new-app에 CLAUDE.md 만들어줘
```

생성되는 CLAUDE.md 예시:
```markdown
# Project: new-app

## Context
이 프로젝트의 작업 기록과 컨텍스트:
- ~/Develop/claude-life/10-projects/personal/new-app.md

세션 시작 시 위 파일을 확인하고 현재 상태를 파악하세요.

## Tech Stack
- Flutter
- Dart

## Commands
- Run: flutter run
- Test: flutter test
- Build: flutter build
```

### 2.4 체크리스트

```
새 프로젝트 시작 시:
[ ] 프로젝트 폴더 생성
[ ] /sync 실행 (자동 등록)
[ ] CLAUDE.md 생성 확인
[ ] 프로젝트 목표 설정 (선택)
```

## 3. 자주 쓰는 명령어

### 3.1 브리핑 & 조회

| 말하면 | 하는 일 |
|--------|---------|
| "오늘 뭐 해?" | 오늘 할 일 브리핑 |
| "브리핑" | 전체 현황 요약 |
| "my-app 현황" | 특정 프로젝트 상태 |
| "진행중인 태스크" | in-progress 태스크 목록 |
| "이번주 뭐 했지?" | 주간 요약 |

### 3.2 기록 & 생성

| 말하면 | 하는 일 |
|--------|---------|
| "OOO 해야해" | 태스크 생성 |
| "OOO 완료" | 태스크 상태 → done |
| "OOO 막혔어" | 태스크 상태 → blocked |
| "아이디어: OOO" | 아이디어 캡처 |
| "배운 것: OOO" | 지식 노트에 추가 |

### 3.3 프로젝트 관리

| 말하면 | 하는 일 |
|--------|---------|
| "/sync" | 프로젝트 동기화 |
| "/context [프로젝트]" | 컨텍스트 상세 로드 |
| "프로젝트 등록해줘" | 수동 프로젝트 등록 |

### 3.4 리뷰

| 말하면 | 하는 일 |
|--------|---------|
| "오늘 정리" | 일일 로그 작성 |
| "주간 회고" | 주간 리뷰 |

## 4. 팁 & 트릭

### 4.1 스마트 트리거 활용

이런 키워드가 포함되면 자동으로 분석 제안:
- 버그, bug
- 문제, issue
- 막힘, stuck
- 에러, error

예:
```
나: 로그인에서 버그 발견했어

Claude: 태스크 생성할게요. 관련 이슈도 분석할까요?
```

### 4.2 프로젝트 폴더에서 바로 기록

프로젝트 작업 중에도 claude-life에 기록 가능:

```
나: 이거 claude-life에 기록해줘: 오늘 인증 모듈 리팩토링 완료
```

### 4.3 Obsidian에서 직접 보기

- 칸반보드: `TaskNotes/Views/kanban-default.base` 열기
- 캘린더: `TaskNotes/Views/calendar-default.base` 열기
- 프로젝트 목록: `10-projects/_index.md` 열기

### 4.4 빠른 캡처

급하게 메모만 하고 싶을 때:

```
나: 인박스: 나중에 확인할 것 - Flutter 3.0 마이그레이션 가이드
```

→ `00-inbox/`에 저장, 나중에 정리

### 4.5 GitHub 연동

프로젝트에 GitHub repo 연결되어 있으면:

```
나: 이 프로젝트 이슈 뭐 있어?

Claude:
#23 Session expires too fast (open)
#21 Login button not responsive (closed)
...
```

## 문제 해결

### TaskNotes API 안 될 때

```bash
# 확인
curl http://127.0.0.1:8090/api/health

# 안 되면: Obsidian 재시작
# 그래도 안 되면: TaskNotes 설정에서 HTTP API 활성화 확인
```

### 프로젝트 컨텍스트 안 불러와질 때

1. 프로젝트 CLAUDE.md에 claude-life 경로 있는지 확인
2. `10-projects/`에 해당 프로젝트 노트 있는지 확인
3. `/sync` 다시 실행

### 태스크가 Obsidian에 안 보일 때

1. Obsidian에서 `20-tasks/` 폴더 확인
2. TaskNotes Views 새로고침 (Cmd+R 또는 뷰 다시 열기)
